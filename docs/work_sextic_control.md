# P3 control experiment: the Degtyarev 4A4+2A1 sextic (order-960 D10-sextic)

**Status: complete for the projective group; affine group not established (TODO, see §8).**

## 0. The question

Degtyarev classified the fundamental groups of all irreducible plane sextics
whose fundamental group has a dihedral quotient `D10` (his notation: `Dn` =
dihedral group *of order* `n`, so `D10` has order 10 — the symmetry group of
a regular pentagon, generated by a 5-cycle rotation and a reflection; this is
what other sources call `D5` in the "`Dn`=order-`2n`" convention). One of his
eight `D10`-sextic families — singularity set `4A4 + 2A1` (four `(2,5)`-cusps,
i.e. four `A4` singular points, plus two nodes, i.e. two `A1` points) — has
**finite** fundamental group `π = π1(P² ∖ B)` of order 960, with maximal
metabelian quotient `π/π'' ≅ D10 × Z3` (order 30).

This is the control experiment for our campaign: a *known, published,
finite* projective fundamental group that (a) genuinely factors through the
pentagon action of `D10` via a dihedral quotient with meridians mapping to
reflections, exactly the setup used in
[`src/n2/local_models.py`](../src/n2/local_models.py), and (b) is not claimed
by anyone to be a Keller-map exceptional curve. Our campaign's working
conjecture is: *"even in this known special-position configuration, the
5-fold irregular dihedral cover has `H1 != Z`."* We computed it. **The
conjecture's literal statement holds — but see the important caveat in §7:
it holds for what turns out to be a somewhat cheap reason (`π` itself is
finite), not necessarily the delicate rank-collapse mechanism the campaign
cares about.**

## 2. Degtyarev's principal theorem (verbatim, §1.2, Theorem 1.2.1)

> Let B be a D10-sextic. Then the group π = π1(P² ∖ B) is finite.
> Furthermore, one has π = D10 ⊗ Z3 with the following two exceptions:
>
> (1) The set of singularities of B is 4A4 ⊕ 2A1: then ord π = 960 and one
> has π″/π‴ = (Z2)⁴ and π‴ = Z2.
>
> (2) The set of singularities of B is A9 ⊕ 2A4 ⊕ A2: then ord π = 21600 and
> π″ is the only perfect group of order 720, see [9].
>
> (In all cases, including the exceptional ones, π/π′ = Z6 and π′/π″ = Z5.)

Our target is exactly exception (1): **4A4 ⊕ 2A1, order 960.** (`⊗` in "D10 ⊗
Z3" is Degtyarev's shorthand for the direct product `D10 × Z3`.)

## 3. The presentation (verbatim, §3.2 and §3.4)

### 3.1 The ambient auxiliary group `Π` and Degtyarev's reduction (§2, §3.2)

Degtyarev does **not** compute `π` directly from meridians of the sextic `B`.
He first shows (Thm. 2.1.1) that every `D10`-sextic `B ⊂ P²` is symmetric:
there is an involution `c : P² → P²` with fixed line `L` and isolated fixed
point `O ∉ B`, such that `B` is the pullback of a very special **trigonal**
curve `B̄` (two type-`A4` points) in the Hirzebruch surface `Σ2 = P²(O)/c`,
under the double cover `P²(O) → Σ2` branched over `L̄ ∪ E` (`E` = exceptional
section). Van Kampen's method applied to the ruling of `Σ2` gives (Prop.
3.2.2):

```
Π := π1(Σ2 ∖ (B̄ ∪ E ∪ L̄))  =  ⟨ α, β, γ, δ  |  m_i = id (i=1..k),  (αβγδ)² = 1 ⟩
```

(`α,β,γ` = meridians of the 3 points where `B̄` meets a generic fiber, `δ` =
meridian of `L̄`; the `m_i` are braid-monodromy relations at singular fibers;
`(αβγδ)²=1` patches the fiber at infinity.)

**Proposition 3.2.5.** `π = π1(P² ∖ B)` is the **kernel** of the
homomorphism `Π/δ² → Z2`, `α,β,γ ↦ 0`, `δ ↦ 1`. So `π` has index 2 in
`Ḡ := Π/⟨⟨δ²⟩⟩`.

### 3.2 §3.4, "The set of singularities 4A4 ⊕ 2A1" (verbatim)

> We choose the generators α, β, δ, γ in a nonsingular fiber between F4 and
> F5. Ignoring the cusp F8, [...] the relations for Π are
>
> (3.4.1)  (αβ)²α = β(αβ)²                              (from the cusp F5),
> (3.4.2)  [β, δ] = [γ, δ] = 1                            (from F4 and F6),
> (3.4.3)  (αδ)² = (δα)²                                  (from F3),
> (3.4.4)  (α⁻¹δαβ)² = (βα⁻¹δα)²                          (from F2),
> (3.4.5)  γ = (α⁻¹δα)⁻¹ β (α⁻¹δα)                        (from the tangent F1),
> (3.4.6)  γ⁻¹αβα⁻¹γ = (αβ)α(αβ)⁻¹                        (from the tangent F7),
> (3.4.7)  (αβδγ)² = 1                                     (patching F∞).
>
> (The relations are simplified using (3.4.2).) The corresponding group π
> given by Proposition 3.2.5 was analyzed using the GAP software package.
> According to GAP, π is an iterated semi-direct product
> Z3 × ((((Z2 × Q8) ⋊ Z2) ⋊ Z5) ⋊ Z2), and the abelian factors of its derived
> series are as stated in Theorem 1.2.1.

**Note on (3.4.7):** it is `(αβδγ)²`, *not* the "generic" `(αβγδ)²` of
Prop. 3.2.2 — `δ` sits between `β` and `γ`. This matches §3.2's remark that
generators are "sometimes reordered by inserting δ ... since δ commutes with
all subsequent generators" — consistent with (3.4.2) (`δ` commutes with `β`
and `γ`) but *not* with `α` ((3.4.3) is a genuine braid relation, not a
commutator).

So, on generators `a=α, b=β, c=γ, d=δ`:

```
Ḡ = Π/δ²  =  ⟨ a, b, c, d  |
    (ab)² a = b (ab)²,
    [b,d] = 1,
    [c,d] = 1,
    (ad)² = (da)²,
    (a⁻¹ d a b)² = (b a⁻¹ d a)²,
    c = (a⁻¹ d a)⁻¹ b (a⁻¹ d a),
    c⁻¹ a b a⁻¹ c = (ab) a (ab)⁻¹,
    (a b d c)² = 1,
    d² = 1  ⟩

π = π1(P² ∖ B)  =  ker( Ḡ ↠ Z2,  a,b,c ↦ 0,  d ↦ 1 )
```

This is the complete, exact input used for every computation below.

## 4. Method (and why we did *not* use `sympy`'s `reidemeister_presentation`)

The task brief flagged a known pitfall from earlier work in this campaign:
`sympy.combinatorics.fp_groups.FpGroup.reidemeister_presentation` **loses the
correspondence between input subgroup generators and the output
presentation's generators** — fatal if you need to track which output
generator is which input meridian (exactly what we need for the
meridian-disk-filling computation in §6). We avoided that code path entirely.

Instead, the group-theory backbone is:

1. **Todd–Coxeter coset enumeration** of the *trivial* subgroup of `Ḡ`
   (`FpGroup.coset_enumeration([], strategy='relator_based',
   max_cosets=200000)`) — this is a completely different, much more
   battle-tested sympy code path, and it gives the **regular permutation
   representation** of `Ḡ`: an explicit, unambiguous permutation of
   `{0,...,1919}` for each of `a,b,c,d`, with *zero* generator-correspondence
   risk (there is nothing to "lose track of" — a permutation *is* the group
   element). This terminated in ≈ 40s and returned exactly **1920** cosets
   (`|Ḡ| = 1920 = 2·960`, matching Degtyarev's `ord π = 960` before we've
   even started analyzing the group — a first correctness check).
2. `π` is then built as the explicit degree-1920 **permutation group**
   generated by the (concrete, already-in-hand) Reidemeister–Schreier
   generators for the index-2 subgroup `Ḡ ⊃ π`, transversal `{1, d}`:
   `π = ⟨ a, b, c, dad⁻¹, dbd⁻¹, dcd⁻¹ ⟩` (all six computed as honest
   permutations, no free-group bookkeeping needed since the ambient
   permutation representation already exists). Order confirmed **960**. ✓
   (These six generators, incidentally, are exactly the six meridian
   generators one would expect from a direct van Kampen presentation of the
   degree-6 curve `B` itself — see §7 for why this is a meaningful
   coincidence, not a red flag.)
3. All further structure (derived series, Sylow subgroups, normal closures,
   the dihedral quotient, the index-5 subgroup, and abelianizations) was
   computed with `sympy.combinatorics.PermutationGroup`'s own methods
   (`derived_subgroup`, `sylow_subgroup`/direct power-map shortcut,
   `normal_closure`, `coset_table`, `abelian_invariants`), all of which act
   on concrete permutations and carry no generator-identity ambiguity.
   `abelian_invariants()` is itself sympy's own from-scratch implementation
   (prime-by-prime power-map decomposition of `G/[G,G]`, source inspected and
   confirmed self-contained) — mathematically the same invariant a
   Reidemeister–Schreier + Smith-normal-form computation would produce, via a
   different, independently-implemented algorithm. For the one place we
   needed the *same* computation relative to a subgroup other than the
   derived subgroup (§6, meridian-disk-filling) we wrote a direct
   generalization of that exact sympy routine (replacing the fixed subgroup
   `H=G'` by an arbitrary normal `M ⊴ G` with `G/M` abelian), reusing its
   verified logic rather than sympy's presentation-rewriting code.

Full scripts are reproduced in §9 (they are short; the total run time for
everything after the initial Todd–Coxeter enumeration is under 3 seconds).

## 6. Finding `ρ : π ↠ D10` and the index-5 subgroup `H`

Since `π/π″ ≅ D10 × Z3` (order 30) and `D10` has no elements of order 3
(`10 = 2·5`), *every* element of order 3 in `π/π″` lies in the `Z3` factor —
so `Z3` is the **unique** subgroup of order 3 in `π/π″`, and the projection
`π/π″ ↠ D10` (killing the `Z3` factor) is the **unique** epimorphism onto
`D10` compatible with the stated metabelian quotient. Concretely:

- `N := π″ · ⟨(a^{ord(a)/3})⟩`, i.e. `π″` extended by any Sylow-3 subgroup
  of `π` (found directly: `a` has order 12 in `π`, so `a⁴` has order 3 — no
  need for sympy's general, and here very slow, `sylow_subgroup()` search).
  **`|N| = 96`**, confirmed **normal** in `π` (all 6 generators' conjugates
  of `N`'s generators checked to lie back in `N`).
- **`[π:N] = 10`**, and `ρ : π ↠ π/N` was verified to be `D10` concretely:
  acting `π` on the 5 cosets of the index-5 subgroup `H` below gives a
  transitive degree-5 permutation image of order 10 with
  `sympy`'s `PermutationGroup.is_dihedral == True`. This *is* `ρ`.
- **Meridians map to reflections, as required.** `a² , b² , c² ∈ N` (i.e.
  `ρ(a), ρ(b), ρ(c)` all have order exactly 2 in `D10`) — and since `D10`
  (`n=5` odd) has *no* order-2 rotations, order-2 elements of `D10` are
  necessarily reflections. Also verified: `a`, `b`, `c` are pairwise
  conjugate inside `π` (found by direct search through `π`'s 960 elements),
  i.e. genuinely a single meridian conjugacy class, as required for an
  irreducible curve.
- **Explicit pentagon action** (`π` acting on the 5 cosets of `H`, points
  labelled `0..4`):
  ```
  ρ(a) = (1 2)(3 4)      -- fixes point 0
  ρ(b) = ρ(c) = (0 1)(2 3)   -- fixes point 4
  ```
  (`ρ(b) = ρ(c)` exactly, as permutations — not a bug: relation (3.4.5) says
  `c` is conjugate to `b` inside `Π`, and it happens that this particular
  quotient map identifies their images; `b` and `c` remain distinct elements
  of `π`.) `⟨ρ(a),ρ(b),ρ(c)⟩` has order 10, is dihedral, transitive on the
  5 points — exactly the "meridians ↦ reflections, pentagon action" spec.
- **`H := ρ⁻¹(Stab_{D10}(0)) = ⟨N, a⟩`.** Since `Stab_{D10}(0) = {1, ρ(a)}`
  (order 2, as `D10` acting on 5 points has order-2 point stabilizers) and
  `a` itself realizes `ρ(a)`, this is literally `N` extended by `a`.
  **`|H| = 192`, `[π:H] = 5`** ✓ — exactly the index-5 subgroup requested.

## 7. `H1` of the index-5 cover

**Projective group.** `H.abelian_invariants() = [2, 2, 2, 3]`, i.e.

```
H1(H)_projective  ≅  Z/2 ⊕ Z/2 ⊕ Z/2 ⊕ Z/3     (elementary-divisor form)
                  ≅  Z/2 ⊕ Z/2 ⊕ Z/6            (invariant-factor form)
                  =  Z^0  ⊕  (finite group of order 24)
```

**Rank 0. `H1 != Z`, confirming the campaign's stated prediction — literally.**

**Meridian-disk-filled variant.** Killing the specific meridian generator `a`
inside `H` (i.e. `(H / ⟨⟨a⟩⟩_H)^{ab}`, computed via `K = H.normal_closure({a})`
(order 48), `M = K·H′` (order 48, so `H′ ⊆ K` already), giving
`[H:M] = 4`, and the generalized-abelian-invariants routine of §4 applied to
`(H, M)`):

```
H1(H)_meridian-a-filled  ≅  Z/2 ⊕ Z/2      (order 4, rank 0)
```

**Important caveat — read before using this as evidence for/against the
campaign's conjecture.** `π` itself has order 960, i.e. is **finite**. *Any*
finite-index subgroup of a finite group is finite, and *any* finite group has
a finite (hence rank-0, torsion-only) abelianization. So `H1(H) != Z` here is
true **for a completely generic, content-free reason** — it would have come
out finite for *any* choice of index-5 subgroup of *any* finite group, not
just this specific dihedral one, and not just because of some delicate
collapse of a would-be-large free rank down to exactly 1 (which is the
*interesting* mechanism the campaign's conjecture is really about, and the
one modeled locally in `src/n2/local_models.py`, where the *ambient* group at
a single cusp is infinite and the question is how much the global relations
manage to cut the rank down). This control, because Degtyarev's sextic
happens to have finite `π1`, cannot by itself distinguish "global relations
collapse the rank all the way to 0 (or to some torsion group)" from "global
relations collapse the rank to exactly 1" — it only tests the weaker, and in
this case automatic, statement "collapse to *something* finite." **I do not
think this control should be read as a nontrivial confirmation of the
rank-collapse mechanism; it is at best a sanity check that the pipeline
(presentation → dihedral quotient → index-5 RS-equivalent computation)
reproduces known, finite, correctly-structured answers on a real example.**
A genuinely discriminating control would need a `D10`-sextic (or a similarly
explicit, citable, non-Keller curve) with **infinite** projective (or affine)
fundamental group and a **known** dihedral quotient — none of Degtyarev's
eight `D10`-sextic families qualify (Theorem 1.2.1: all eight are finite).
Flagging this as a finding for the campaign, not burying it.

## 8. TODO / not established

  produces `π = π1(P² ∖ B)` via the auxiliary `Σ2` double-cover machinery of
  §3.1, *not* via a direct van Kampen presentation of `B` in its own generic
  affine chart. The relation `(αβδγ)² = 1` (3.4.7) that appears in `Ḡ`
  patches the fiber at infinity of the auxiliary ruled surface `Σ2` — it is
  **not** the meridian-at-infinity relation of the actual ambient `P²`
  containing `B`, so simply "dropping one relator" from `Ḡ` does **not**
  correspond to passing to `C² ∖ B`. Recovering `π1(C² ∖ B)` rigorously would

## 9. Scripts (for reproducibility)

All computations were run with `sympy` 1.14.0, Python 3.10, on Windows
(no GAP/Sage/Magma available on this machine or in its WSL Ubuntu install —
checked and confirmed absent; `apt-get install gap` requires interactive
sudo, not available non-interactively). Three short scripts, run in sequence
(outputs quoted inline in §5–§6 above):

**Script 1 — build `Ḡ`'s regular permutation representation via Todd–Coxeter**
(≈ 40s):

```python
from sympy.combinatorics.free_groups import free_group
from sympy.combinatorics.fp_groups import FpGroup
from sympy.combinatorics import Permutation, PermutationGroup

F, a, b, c, d = free_group("a, b, c, d")
R = [
    (a*b)**2 * a * (b*(a*b)**2)**-1,           # 3.4.1
    b*d*b**-1*d**-1,                           # 3.4.2a  [b,d]=1
    c*d*c**-1*d**-1,                           # 3.4.2b  [c,d]=1
    (a*d)**2 * ((d*a)**2)**-1,                 # 3.4.3
    (a**-1*d*a*b)**2 * ((b*a**-1*d*a)**2)**-1, # 3.4.4
    c * ((a**-1*d*a)**-1 * b * (a**-1*d*a))**-1,  # 3.4.5
    (c**-1*a*b*a**-1*c) * ((a*b)*a*(a*b)**-1)**-1,  # 3.4.6
    (a*b*d*c)**2,                              # 3.4.7 (note order a,b,d,c)
    d**2,                                      # delta^2 = 1
]
G = FpGroup(F, R)
C = G.coset_enumeration([], strategy='relator_based', max_cosets=200000)
C.compress(); C.standardize()
gens = G.generators
n = len(C.table)
images = [[C.table[i][2*gens.index(g)] for i in range(n)] for g in gens]
pa, pb, pc, pd = [Permutation(im) for im in images]
Pbar = PermutationGroup([pa, pb, pc, pd])
assert Pbar.order() == 1920
```

**Script 2 — `π`, derived series, dihedral quotient, index-5 subgroup, `H1`:**

```python
pa2, pb2, pc2 = pd*pa*pd**-1, pd*pb*pd**-1, pd*pc*pd**-1
PI = PermutationGroup([pa, pb, pc, pa2, pb2, pc2])
assert PI.order() == 960                                   # = pi

Dp   = PI.derived_subgroup();  Dpp = Dp.derived_subgroup()
Dppp = Dpp.derived_subgroup()
# orders 160, 32, 2 ; abelian_invariants [5], [2,2,2,2], [2]  -- match Thm 1.2.1

gen3 = pa**(pa.order()//3)                                  # order-3 element (Sylow-3, fast path)
P3 = PermutationGroup([gen3])
N = PermutationGroup(list(Dpp.generators) + list(P3.generators))
assert N.order() == 96                                      # pi'' . Z3, normal, index 10

H = PermutationGroup(list(N.generators) + [pa])
assert H.order() == 192 and PI.order()//H.order() == 5

print(H.abelian_invariants())                                # [2, 2, 2, 3]  <-  H1(H) projective

# pentagon action (rho), read off from the coset table of H in PI:
table = PI.coset_table(H)                                    # 5 rows
def gen_perm(idx): return Permutation([table[i][2*idx] for i in range(5)])
gi = list(PI.generators)
rho_a, rho_b, rho_c = gen_perm(gi.index(pa)), gen_perm(gi.index(pb)), gen_perm(gi.index(pc))
Im = PermutationGroup([rho_a, rho_b, rho_c])
assert Im.order() == 10 and Im.is_dihedral and Im.is_transitive()
```

**Script 3 — meridian-disk-filled variant (generalized `abelian_invariants`):**

```python
from sympy import primefactors, multiplicity

def abelian_invariants_rel(Gtop, Mfixed):
    """Invariant factors of Gtop/Mfixed (Mfixed normal, Gtop/Mfixed abelian).
    Direct generalization of PermutationGroup.abelian_invariants, which is
    the special case Mfixed = Gtop.derived_subgroup()."""
    gns = list(Gtop.generators); inv = []; G = Gtop
    Mgens = list(Mfixed.generators)
    for p in primefactors(Gtop.order()//Mfixed.order()):
        ranks = []
        while True:
            pows = [g**p for g in gns if not Mfixed.contains(g**p)]
            K = PermutationGroup(Mgens + pows) if pows else Mfixed
            r = G.order()//K.order()
```
